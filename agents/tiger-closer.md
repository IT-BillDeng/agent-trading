# Tiger Closer Agent

## 职责
每个市场收盘后生成总结报告，包括：
1. 当日执行概况
2. 信号质量分析
3. 盈亏汇总
4. 明日关注标的

## 触发时机
- **港股收盘**：16:00 HKT (08:00 UTC)
- **美股收盘**：16:00 ET (20:00 UTC / 次日 04:00 北京时间)

## 数据来源
- `/api/engine` - 最近执行周期
- `/api/signals` - 信号数据
- `/api/account` - 账户状态
- `/api/positions` - 持仓信息
- `/api/orders` - 订单记录
- `/api/audit` - 审计日志

## 输出格式

### 收盘总结模板
```
📊 Tiger 收盘总结 [日期] [市场]

═══ 执行概况 ═══
• 运行周期: X 次
• 信号数量: BUY X / EXIT X / HOLD X
• 订单执行: 成功 X / 失败 X

═══ 账户状态 ═══
• 净值: $XXX,XXX.XX
• 今日盈亏: $X,XXX.XX (X.XX%)
• 持仓: X 只

═══ 信号质量 ═══
• 触发信号: [列表]
• 风控拦截: X 项

═══ 明日关注 ═══
• [标的1]: [原因]
• [标的2]: [原因]

═══ 风险提示 ═══
• [如有异常/风险，列出]
```

## 处理逻辑

1. 检查市场是否刚收盘（避免重复发送）
2. 读取当天所有执行周期数据
3. 汇总信号、订单、盈亏
4. 生成总结报告
5. 通过 Telegram 发送给先生
6. 写入历史记录

## 告警抑制
- 每个市场每天只发送一次收盘总结
- 如有异常，在总结中标注
