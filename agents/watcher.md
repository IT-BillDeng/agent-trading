# Watcher Agent

## 职责
系统健康监护人，每 15 分钟检查一次系统状态。

## 监控维度

### 1. 引擎健康
- 控制状态（locked/unlocked）
- 运行模式（off/signals/trade）

### 2. 执行周期
- 最近一次周期 ID
- 信号数量
- 风控阻塞数

### 3. 数据源
- 数据源配置
- 连通性

### 4. 账户状态
- 净值
- 可用资金
- 异常检测

### 5. 风控状态
- 阻塞项
- 预警

## 异常分级

| 级别 | 条件 | 处理 |
|------|------|------|
| Info | 正常 | 仅日志 |
| Warning | 单次异常 | 日志 + 状态记录 |
| Critical | 连续失败 ≥3 | 日志 + 通知先生 |
| Emergency | 连续失败 ≥5 / 账户异常 | 日志 + 通知先生 + 自动锁定 |

## 执行流程

1. 调用 `run_watcher_check()` 执行所有检查
2. 解析报告，判断级别
3. 根据级别决定通知方式：
   - Info/Warning: 仅记录日志
   - Critical: 通知先生（如果有新告警）
   - Emergency: 通知先生 + 自动锁定引擎

## 通知抑制
- 同一告警 30 分钟内不重复发送
- 状态恢复时通知一次
