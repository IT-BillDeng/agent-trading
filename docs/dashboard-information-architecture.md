# Dashboard 信息架构方案

## 目标

把当前 Dashboard 从“信息堆叠页”收口成三类页面：

- `/` 总览页：先回答“现在要不要介入”
- `/strategy` 策略页：回答“策略在干什么、为什么这么做”
- `/logs` 日志页：回答“系统为什么异常、最近发生了什么”

原则：

- 首页只放当前态，不放历史明细
- 一张卡只回答一个问题
- 摘要优先，明细折叠或下沉到二级页
- 颜色表达紧急程度，文字表达原因

---

## 当前问题

### 1. 首页混了太多层级

当前首页同时放了：

- 账户总览
- 持仓与行情
- 个股分析
- 最近订单
- Newswire
- 自选管理
- Engine 运行结果
- 回测报告
- 风控
- 通知预览
- API 配置
- 控制面板
- 系统状态

这导致首页既像“交易看板”，又像“调试控制台”，还像“配置页”。

### 2. 状态、历史、配置混杂

例如：

- `🛡️ 风控` 显示的是执行前风险审查结果
- `🔧 Engine 运行结果` 显示的是最近一轮 cycle 快照
- `📋 最近订单` 显示的是 broker 当前订单列表
- `🔑 API 配置` 和 `🎛️ 控制面板` 又是操作入口

这些都在首页，会让用户不清楚：

- 这是当前状态
- 还是历史结果
- 还是可以修改的配置

### 3. 默认展示粒度过细

像：

- 风控逐条 decision
- 最近订单逐笔列表
- 个股分析明细表

本身是有用的，但不适合作为首页默认第一视图。

---

## 建议页面职责

## 1. `/` 总览页

职责：一眼看清系统现在是否正常、是否在交易窗口、是否需要介入。

### 首页建议保留的模块

#### A. 账户总览

保留：

- 总资产
- 今日盈亏
- 预估现金额
- 持仓总价值
- 未实现盈亏

这是首页最重要的“结果层”。

#### B. 市场与系统状态

合并当前分散的状态信息，做成一张“状态条卡片”：

- 今日是否为交易日
- 当前市场时段：盘前 / 盘中 / 盘后 / 休市
- trading mode：off / signals / trade
- 最近一轮 cycle 是否成功
- 当前是否存在异常/阻塞

这张卡应该回答：

- 今天该不该交易
- 系统现在是不是正常运行

#### C. 异常与阻塞摘要

建议由以下来源汇总：

- 风控阻塞
- watcher 异常
- scheduler / dashboard service error
- 关键 agent 缺少最新产物

首页只显示：

- 异常数量
- 阻塞数量
- Top 3 原因

不要直接平铺全部明细。

#### D. 持仓

保留，但作为首页主要业务面板之一。

原因：

- 它是当前账户最直接的状态
- 比历史订单和个股分析更接近“现在发生了什么”

#### E. Newswire 摘要

首页不要保留整块完整新闻流。

建议改成摘要卡：

- 最新生成时间
- high-importance 条数
- 影响标的 Top N
- 一键跳转详细新闻页或展开

#### F. 个股分析

保留在首页。

原因：

- 它直接回答“今天哪些标的值得看”
- 是交易决策主视图的一部分
- 与持仓、最近订单属于同一层“业务结果”

#### G. 最近订单

保留在首页。

原因：

- 它直接回答“刚刚发生了什么”
- 可以快速判断 EXIT / BUY 是否已经形成 adapter order record
- 对日常盯盘非常关键

### 首页建议移走的模块

移到二级页或折叠区：

- `🛡️ 风控` 详细列表
- `🔧 Engine 运行结果`
- `📈 回测报告`
- `🔔 通知预览`
- `🔑 API 配置`
- `🎛️ 控制面板`
- `📝 自选管理`

这些不是“首页一眼判断当前状态”必需的。

---

## 2. `/strategy` 策略页

职责：成为真正的策略工作台。

### 建议承担的内容

- 策略开关与启用状态
- 每只股票在什么策略下产生了什么信号
- 最新策略调整
- Strategist 调整时间线
- 风控审查结果（针对信号）
- 回测与策略迭代摘要

### 为什么风控也更适合部分下沉到这里

当前 `🛡️ 风控` 显示的是：

- 信号执行前审查
- preview blockers

它本质是“策略到执行之间”的中间层，不是账户总览层。

因此建议：

- 首页只放风控摘要
- `/strategy` 放执行前风控明细

---

## 3. `/logs` 日志页

职责：系统诊断与运行排障。

继续承载：

- latest snapshots
- agents status
- audit/service/legacy logs
- 日志筛选与详情

不要再把这套内容回流到首页。

---

## 首页推荐新布局

## 顶部：结果层

### 第一行

- 总资产
- 今日盈亏
- 今日是否交易日 / 市场状态

### 第二行

- 预估现金额
- 持仓总价值
- 未实现盈亏

## 中部：当前态

### 左侧 2/3

- `📊 持仓`
- `💰 个股分析`

### 右侧 1/3

- `系统状态 / 异常摘要`
- `📋 最近订单`

## 下部：信息流

- `📰 Newswire`

## 底部：快捷跳转

- 跳转到策略页
- 跳转到日志页
- 跳转到配置/控制页

首页不再承载完整工作流。

---

## 风控块建议重构

当前标题：`🛡️ 风控`

建议改成：`🛡️ 执行前风控`

### 首页摘要版

只显示：

- `可继续执行：x / y`
- `执行阻塞：n`
- `主要原因：market_not_regular_session / no_position_to_exit / active_order_exists`

### 策略页明细版

显示：

- 每个 symbol/action 的 decision
- preview blockers
- cycle_id

这样能同时满足：

- 首页快速判断
- 策略页精细排查

---

## 最近订单建议

`📋 最近订单` 建议继续保留在首页，但保持轻量展示。

首页只回答：

- 刚刚发生了什么
- 最近是否真的发出了 BUY / SELL / EXIT 对应订单

更深的执行诊断再放到 `/strategy` 或未来的 `/execution`。

---

## 配置与控制建议

这些应逐步独立：

- API 配置
- 风控参数
- Trading mode
- 手动刷新 / 管理动作

建议最后形成：

- `/settings`
- 或 `/ops`

如果短期不想加新页，也至少放到首页底部折叠区，不要和业务信息并列。

---

## 推荐实施顺序

### 第一步

重做首页，不改数据源：

- 首页只保留账户总览、持仓、个股分析、最近订单、系统状态、异常摘要
- 风控改为摘要

### 第二步

把 `/strategy` 扩成真正工作台：

- 接入执行前风控明细
- 接入最近执行动作与策略视角的订单说明

### 第三步

把配置和控制从首页抽离：

- API 配置
- 控制面板
- 风控参数

---

## 一句话原则

首页回答：

- 现在系统是否正常
- 今天是否值得关注
- 是否需要我介入

策略页回答：

- 策略现在在干什么
- 哪些股票触发了什么
- strategist 最近改了什么

日志页回答：

- 为什么异常
- 最近发生了什么
